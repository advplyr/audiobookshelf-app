## 4. Conflict resolution

### 4.1 `components/app/AudioPlayer.vue` — 3 conflict zones

**Zone 1 — template lines.** Upstream `8be08702` renamed `isLoading` → `showLoadingState` on ~10 template lines; our `14034b87` added `tabindex="0"` and `@keydown.enter.prevent[.stop]` handlers to the same lines.

Resolution: **combine both edits per line.** Take upstream's `:class="showLoadingState ? ... : ..."` rename in every binding, AND keep our `tabindex` + `@keydown.enter` attributes. Affected controls: `jumpChapterStart` span, `jumpBackwards` div, `play-btn` div, `jumpForward` div, `jumpNextChapter` span. The progress-track div and the play-button inner `<span v-if="!isLoading">` get only upstream's rename (we don't touch them).

**Zone 2 — import block.** End state, in order: `import { Dialog } from '@capacitor/dialog'`, `import { KeepAwake } from '@capacitor-community/keep-awake'` (ours), `import { getAverageColorFromCoverUrl } from '@/utils/coverAverageColor'` (upstream's util replacement for `FastAverageColor`). Drop the old `FastAverageColor` import.

**Zone 3 — `coverImageLoaded` method body + new data/computed/methods.** Take upstream verbatim. The new `isCheckingServerProgress` data prop, `showLoadingState()` computed, `setIsCheckingServerProgress(value)` method all land as-is. Our `updateKeepAwake`, `collapseFullscreen` TV-close branch, History menu hide, and `closePlayback` keepAwake call are in untouched regions and ride along.

### 4.2 `pages/item/_id/index.vue` — 2 conflict zones

**Zone 1 — cover element block.** Upstream removed `@imageLoaded="coverImageLoaded"` from `<covers-book-cover>` (the new util fetches the cover itself). We added `id="item-cover" tabindex="0" @click @keydown.enter` to the wrapping `<div>`. Combine:

```vue
<div id="item-cover" class="relative" tabindex="0"
     @click="showFullscreenCover = true"
     @keydown.enter="showFullscreenCover = true">
  <covers-book-cover :library-item="libraryItem" :width="coverWidth"
                     :book-cover-aspect-ratio="bookCoverAspectRatio" no-bg raw />
  <div v-if="!isPodcast" class="absolute bottom-0 left-0 h-1 z-10 box-shadow-progressbar"
       :class="userIsFinished ? 'bg-success' : 'bg-yellow-400'"
       :style="{ width: coverWidth * progressPercent + 'px' }"></div>
</div>
```

**Zone 2 — import + `coverImageLoaded` method body.** Take upstream verbatim. After the merge, `coverImageLoaded` becomes an unreferenced method in this file (its only caller — the `@imageLoaded` listener — was removed). Harmless and symmetric with upstream; leave it. If maintainer review flags it later, drop it then.
